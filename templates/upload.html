DiffCompare tool for NER evaluation

Upload a .zip file with the following structure:
f.zip
├── gold/
└──── f1.ann/xml
└──── f2.ann/xml
└──── f3.ann/xml
├── to_eval/
└──── f1.ann/xml
└──── f2.ann/xml
└──── f3.ann/xml
├── text/
└──── f1.txt
└──── f2.txt
└──── f3.txt
  • gold/ directory can contain both .xml and .ann files
  • to_eval/ directory can contain both .xml and .ann files
  • it is important that the files to be evaluated and their corresponding gold standards and text files have the same name
  • it is important that xml files (excluding NER tags) align with their text correspondents
  • if xml files (excluding NER tags) don't align with their text correspondents, you will be notified about the exact lines that need to be aligned
  • if you don't get visualization for desired entity, it means it's not present in annotated files
  • if you don't get proper output where both gold and to_eval files are of ann type, characters in ann probably don't align with text files
